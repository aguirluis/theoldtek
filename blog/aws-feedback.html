Building a Serverless Feedback Form with AWS and GitHub Pages

Published: 2025

I wanted a simple way to collect feedback without running a server, managing a database, or worrying about scaling. That led me to a fully serverless stack: AWS Lambda, API Gateway, and DynamoDB, with a static frontend hosted on GitHub Pages.

Architecture at a glance

The flow looks like this:

  • The user fills out a feedback form on a static HTML page.
  • The browser sends a POST request to an AWS API Gateway endpoint.
  • API Gateway invokes a Lambda function.
  • The Lambda function writes the feedback into a DynamoDB table called FeedbackTable.
  • The Lambda returns a JSON response, which the frontend shows as a success message.

Backend: Lambda + DynamoDB

The Lambda function is written in Node.js 22.x and uses the AWS SDK v3. It parses the incoming request body, generates a unique feedbackId, and stores the data in DynamoDB.

API Gateway: Exposing the Lambda as HTTP

I created a REST API in API Gateway with a POST /feedback method integrated with the Lambda using Lambda proxy integration. I also enabled CORS and tested the method directly in the console until I consistently got a 200 OK with a JSON success message.

Frontend: Static HTML on GitHub Pages

The page you're on now, aws-feedback.html, is a simple HTML form that sends a JSON payload to the API Gateway endpoint using the Fetch API. Because the backend already handles CORS and returns clean JSON, the frontend code stays very small and focused.

Why this approach works well

  • No servers to manage: Lambda and DynamoDB are fully managed.
  • Scales automatically: It can handle spikes in traffic without extra work.
  • Cost-effective: You only pay for what you use.
  • Simple frontend: A single static HTML file can power the entire experience.

If you submit the form below, your feedback is stored in DynamoDB in real time. This small project ended up being a great way to learn how AWS services fit together in a real, end-to-end workflow.

Building a Serverless Feedback Form with AWS and GitHub Pages

Published: 2025

After completing my AZ‑900 certification, I wanted a new challenge—something practical that would push me to apply my cloud fundamentals in a real project. That’s when I decided to build my AWS Feedback App, a hands‑on project designed to strengthen my understanding of cloud‑native development by creating a fully functional, serverless application on AWS.

While my experience as a Systems Administrator has given me plenty of exposure to deploying applications in local environments, I hadn’t yet taken a deep dive into building and deploying solutions in the cloud. This project became the perfect opportunity to bridge that gap and gain real experience with AWS services, architecture patterns, and modern application workflows.

After passing my AZ‑900, I wanted a simple way to collect feedback without running a server, managing a database, or worrying about scaling. That led me to a fully serverless stack: AWS Lambda, API Gateway, and DynamoDB, with a static frontend hosted on GitHub Pages.

Architecture at a Glance

The full workflow looks like this:


+------------------+        +---------------------+        +----------------------+
|  GitHub Pages    | -----> |   API Gateway       | -----> |     AWS Lambda       |
|  Static HTML/JS  |  POST  |  /feedback endpoint |  JSON  |  Node.js 22.x        |
+------------------+        +---------------------+        +----------+-----------+
                                                                      |
                                                                      v
                                                          +----------------------+
                                                          |   DynamoDB Table     |
                                                          |   FeedbackTable      |
                                                          +----------------------+
  • User submits a form on a static HTML page.
  • Browser sends a POST request to API Gateway.
  • API Gateway triggers a Lambda function.
  • Lambda writes the feedback into DynamoDB.
  • A JSON response is returned to the frontend.

Backend: Lambda + DynamoDB

The Lambda function is written in Node.js 22.x using AWS SDK v3. It validates the input, generates a unique feedbackId, and stores the record in DynamoDB.

Lambda Function Code (Node.js 22.x)


// index.mjs
import { DynamoDBClient, PutItemCommand } from "@aws-sdk/client-dynamodb";

const db = new DynamoDBClient({ region: "us-east-1" });

export const handler = async (event) => {
  const body = JSON.parse(event.body);

  const item = {
    feedbackId: { S: `fb-${Date.now()}` },
    name: { S: body.name },
    email: { S: body.email },
    message: { S: body.message },
    timestamp: { S: new Date().toISOString() }
  };

  await db.send(new PutItemCommand({
    TableName: "FeedbackTable",
    Item: item
  }));

  return {
    statusCode: 200,
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ success: true })
  };
};

DynamoDB Table Structure


Table Name: FeedbackTable

Partition Key: feedbackId (String)

Attributes:
- name (String)
- email (String)
- message (String)
- timestamp (String)

API Gateway: Exposing the Lambda as HTTP

API Gateway exposes the Lambda function through a POST /feedback endpoint. I enabled CORS, configured Lambda Proxy Integration, and tested the method until I consistently received a 200 OK response.

Frontend: Static HTML on GitHub Pages

The frontend is intentionally lightweight — a single HTML file with a form and a small JavaScript snippet that sends the data to API Gateway.

Frontend Fetch Code


fetch("https://u83asanvo3.execute-api.us-east-1.amazonaws.com/prod/feedback", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    name: nameInput.value,

  })
})
.then(res => res.json())
.then(() => alert("Feedback submitted!"))
.catch(() => alert("Something went wrong."));

Why This Approach Works Well

  • No servers to manage: Lambda and DynamoDB are fully managed.
  • Scales automatically: Handles traffic spikes without configuration.
  • Cost-effective: Stays within the AWS free tier for low traffic.
  • Simple frontend: A single static HTML file powers the entire experience.

Lessons Learned

  • API Gateway CORS errors are the #1 issue when connecting static sites to AWS APIs.
  • DynamoDB is extremely fast for small payloads — perfect for logging and feedback systems.
  • Lambda cold starts are negligible for lightweight Node.js functions.
  • Testing API Gateway directly in the console saves a lot of debugging time.
  • Serverless apps force you to think in events, not servers — a valuable mindset shift.

Future Improvements

  • Add reCAPTCHA or hCaptcha to prevent spam.
  • Implement email notifications using SES.
  • Add a small admin dashboard to view feedback entries.
  • Use API Gateway throttling or AWS WAF for rate limiting.
  • Replace REST API with HTTP API for lower cost and simpler routing.

Cost Breakdown

For small-scale usage, this entire system runs inside the AWS Free Tier:

  • Lambda: First 1M requests/month are free.
  • API Gateway: First 1M REST API calls/month are free.
  • DynamoDB: Free tier includes 25GB storage + 25 RCU/WCU.
  • GitHub Pages: Free static hosting.

Even outside the free tier, this architecture costs only a few cents per month unless you're handling thousands of submissions daily.
