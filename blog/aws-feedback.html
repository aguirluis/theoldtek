Building a Serverless Feedback Form with AWS and GitHub Pages

Published: 2025

While studying for the AWS Associate certification, I wanted a hands-on project that used real AWS services without requiring servers or complex infrastructure. The result was a fully serverless feedback-collection app powered by:

  • AWS Lambda
  • API Gateway
  • DynamoDB
  • GitHub Pages for the frontend

Step 1: Build the Frontend (Free)

  • Simple HTML/JS form
  • Hosted on GitHub Pages
  • Sends POST requests to your cloud API

Step 2: Add a Database

  • Store form submissions
  • Use DynamoDB, Firestore, or Cosmos DB free tier

Step 3: Add an Event Trigger

Choose one:

  • Send an email (SES, SendGrid)
  • Write logs to CloudWatch
  • Trigger another function
  • Generate a summary report

Step 4: Add Monitoring

  • Track number of submissions
  • Track errors
  • Create a dashboard

Step 5: Add CI/CD

  • GitHub Actions deploys your function automatically
  • Another workflow deploys your frontend

Architecture Overview

  • User fills out a feedback form on a static HTML page.
  • The browser sends a POST request to an API Gateway endpoint.
  • API Gateway invokes a Lambda function.
  • Lambda writes the feedback into a DynamoDB table (FeedbackTable).
  • The Lambda returns a JSON response displayed by the frontend.

Backend: Lambda + DynamoDB

The Lambda function (Node.js 22.x) uses AWS SDK v3 to parse the request, generate a unique feedbackId, and store the data in DynamoDB.

API Gateway: Exposing the Lambda

I created a REST API with a POST /feedback method using Lambda proxy integration. CORS was enabled, and I tested the method until it consistently returned 200 OK.

Frontend: Static HTML on GitHub Pages

The page you're reading is a simple HTML form that sends JSON to the API Gateway endpoint. Because the backend handles CORS and returns clean JSON, the frontend stays lightweight.

Why This Approach Works Well

  • No servers to manage: Lambda and DynamoDB are fully managed.
  • Scales automatically: Handles traffic spikes effortlessly.
  • Cost-effective: Pay only for what you use.
  • Simple frontend: A single static HTML file powers the experience.
